# 切换到测试页面验证环境

## 目的

如果您在模拟器中看不到内容，可以切换到一个简化的测试页面来验证环境是否正常。

---

## 快速切换方法

### 步骤 1：修改配置文件

编辑 `mobile/package.json`，找到 `main` 字段：

```json
{
  "main": "app/app.ts",    ← 改为下面这行
  "main": "app/app-test.ts"  ← 使用测试版本
}
```

或者直接运行以下命令：

```powershell
cd mobile
(Get-Content package.json) -replace '"main": "app/app.ts"', '"main": "app/app-test.ts"' | Set-Content package.json
```

### 步骤 2：重新运行

```powershell
npm run android
```

### 步骤 3：查看结果

**如果您能看到测试页面**（带有 ✅ 标记和环境信息），说明：
- ✅ Java 17 环境完全正常
- ✅ NativeScript 工作正常
- ✅ Vue 3 渲染正常
- ✅ 问题出在原始 LoginPage 代码

**如果仍然看不到**，可能是：
- 模拟器窗口问题（查找窗口）
- 需要更多等待时间
- 或其他系统问题

---

## 恢复原始版本

测试完成后，恢复原始配置：

```powershell
cd mobile
(Get-Content package.json) -replace '"main": "app/app-test.ts"', '"main": "app/app.ts"' | Set-Content package.json
npm run android
```

---

## 测试页面功能

测试页面会显示：
1. ✅ Java 17 环境状态
2. ✅ NativeScript Runtime 版本
3. ✅ Vue 3 框架状态
4. 一个可点击的按钮测试交互

**如果都能看到和点击，说明环境 100% 正常！**

- **能看到测试页面** = Java 17 环境完美，继续开发原始应用
- **看不到任何内容** = 可能是模拟器显示问题，需要进一步排查
